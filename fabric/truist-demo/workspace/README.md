## Re-applying the Organization

If the workspace needs to be rebuilt or reorganized:

```powershell
# Prerequisites: az login --tenant 72f988bf-86f1-41af-91ab-2d7cd011db47
.\setup-workspace.ps1          # create folders + move all items
.\setup-workspace.ps1 -WhatIf  # dry-run preview
```

To restore the task flow:
1. Open the [Tarpon workspace](https://msit.powerbi.com/groups/09fd4407-ad4f-42a6-92cd-c69bccd5daa1/list?experience=power-bi)
2. Task flow canvas → Import → select `Tarpon-Data-Platform-task-flow.json`
3. Assign items to tasks per the table above (clip icon on each task card)

## API Notes

- **Correct move body field:** `targetFolderId` (not `destinationFolderId` — the wrong name is silently ignored with HTTP 200)
- **Child items** (SQLEndpoint, KQLDatabase) cannot be moved independently — they auto-follow their parent Lakehouse/Eventhouse
- **Task flow JSON** does not support item assignments in the export/import format — assignments must be set in the UI after import
